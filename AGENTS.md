# Agent Instructions for This Repository

This repository is public-facing. Keep it safe, boring, and interoperable.

## Do

- Document only public Roop MCP URLs and public Roop documentation.
- Keep setup instructions vendor-neutral where possible.
- Use local environment variables for authenticated access examples.
- Make anonymous/free-limit handoff explicit.
- Tell agents to cite MCP-returned public references and disclose uncertainty.
- Preserve the message that LegalSearch enables interoperability, while Roop's native platform is the integrated home for sustained legal workflows.

## Do Not

- Add non-public Roop infrastructure, hostnames, paths, operational details, source-code internals, private technical names, or privileged endpoints.
- Add raw secrets, example tokens that look real, passwords, API keys, or bearer-token literals.
- Recommend bypassing rate limits, authentication, robots guidance, packet caps, or MCP boundaries.
- Claim the MCP provides legal advice.
- Claim a pricing advantage unless the pricing has been approved and publicly published.
